
ctrl+d - копия текущей строки или выделенного фрагмента кода
ctrl+/ - коментить вибрану частину коду або строку

pytest  запуск с терминала всех тестов в файле с без визуализациии результатов
pytest -v  запуск с терминала всех тестов в файле с визуализацией результатов
pytest -v -s test_demo.py - запуск одного файла с тестами
pytest -v -s test_demo.py::test_demo2 -  - запуск указаного теста в указаном файле с тестами (файл.py::тест)

pip install -r requirements.txt  установка всех модулей из файла requirements.txt
pip freeze requirements.txt - добавляет всё установленное в requirements.txt

git status - посмотреть на то что просится уйти на гіт
git add . – додає всі зміни (нові файли, модифікації та видалення) в поточній директорії та піддиректоріях для наступного коміту. Це команда, яку часто використовують, щоб підготувати всі зміни до коміту.
git commit -m "tests ready" створює новий коміт з повідомленням "tests ready". Це означає, що всі проіндексовані (git add) зміни зберігаються в історії комітів.
git push відправляє локальні коміти у віддалений репозиторій (наприклад, на GitHub). Вона синхронізує ваші зміни з віддаленою гілкою.

pytest --junitxml=./logs/result.xml - записує результат у файл за вказаним путем
pytest --alluredir=allure-results - вказує, куди зберігатимуться результати тестів для Allure (у папку allure-results).
allure serve allure-results - генерує звіти в алюр

pytest --alluredir=allure-results
